# Kotlin 迁移规范：Java 到 Kotlin 转换

## 目标

将项目中所有 Java 文件迁移到 Kotlin，保持功能完全一致，遵循 Kotlin 语言惯用法，同时确保与 Java 代码的互操作性。

---

## 核心要求

### 迁移范围

- 将项目中所有 `.java` 文件转换为 `.kt` 文件
- 保持原有包结构和类名不变
- 确保所有业务逻辑、算法、行为与原 Java 代码完全一致

### 空安全处理

| Java 注解 | Kotlin 类型 |
|-----------|-------------|
| `@Nullable`、`@CheckForNull` | `String?`（可空类型） |
| `@NonNull`、`@NotNull` | `String`（非空类型） |
| 无注解 | 根据运行时可能为 `null` 的实际情况判断 |

**判断原则：**
- 字段/方法有默认值或保证非空 → 非空类型（如 `String`）
- 未明确保证、来自外部数据、可能缺失 → 可空类型（如 `String?`）
- 集合元素如需可空，使用 `List<String?>` 而非可空集合

### 互操作性要求

- **工具类和静态方法**：转换后尽量保持与 Java 相同的调用方式，使用 `@JvmStatic`
- **重载方法**：优先使用 `@JvmOverloads` 生成多个构造函数/方法重载
- **对外公开的类**：对于子模块中对外公开的类，声明为 `open` 以允许被继承；对于不确定是否需要被继承的类，优先声明为 `open`

---

## 迁移步骤

- 识别模块间的依赖，优先转换底层/被依赖少的类
- 记录哪些类被 Java 代码调用（需保持 `open` 或 `@JvmStatic`）
- 复制 Java 代码逻辑，转换为 Kotlin 语法
- 根据空安全规则，将 Java 类型转换为对应的 Kotlin 可空或非空类型
- 添加必要的 `open` 关键字
- 添加 `@JvmStatic` 或 `@JvmOverloads` 保持互操作性
